# Xunxiashi Question Set

This file contains the final 30 user-facing questions.

These questions are written for novice users.
They must be asked in plain language.

Do not dump all 30 questions by default.
Ask them progressively unless the user explicitly requests the full set.

## What Is Not Asked

These are built-in defaults and should not be presented as feature-existence questions:

- startup recovery exists
- `/new` short-term recovery exists
- "remembered" claims require a successful write

You may still ask about user preference for how these behaviors should feel.

## Stage 1: Build the Brain

Use these first when the workspace is new, partial, or being rebuilt.

1. How should I address you?

2. What kind of work do you mainly do? Say it in one plain sentence.
   Hints:
   - screenwriter
   - ecommerce operator
   - salesperson
   - product manager
   - teacher

3. What is the main kind of thing you most want this shrimp to help you with long-term?
   Hints:
   - organizing
   - writing
   - research
   - reminders
   - review and summarization

4. Should this shrimp serve only you by default, or also certain approved people?
   Hints:
   - only me
   - approved people too
   - unknown people should wait for my confirmation

5. If a stranger, unauthorized person, or unclear source contacts it, what should it do?
   Hints:
   - ignore
   - ask me first
   - reply with an authorization notice

6. For risky actions like sending, editing files, submitting content, or operating on your behalf, what should it do by default?
   Hints:
   - ask every time
   - ask only for high-risk actions
   - only advise, do not act directly

7. What kinds of decisions must it never make for you?
   Hints:
   - pricing
   - commitments
   - external promises
   - money
   - personnel decisions

8. If it does not know the answer or cannot find reliable information, how should it say that?
   Hints:
   - admit it clearly
   - say it is uncertain and needs to check more
   - only answer the confirmed part

9. What kind of work companion should this shrimp feel like overall?
   Hints:
   - reliable assistant
   - calm strategist
   - diligent executor
   - clever helper
   - sharp but useful partner

10. What kind of wording, tone, or reply style do you least want to see from it?
   Hints:
   - customer-service tone
   - obvious AI tone
   - too much fluff
   - too stiff
   - too oily
   - pretending to know everything

## Stage 2: Train the Brain

Use these during real work and during progressive training.

11. What are the three kinds of things you deal with most often every day?
   Hints:
   - writing
   - revising
   - communicating
   - researching
   - organizing ideas
   - tracking tasks

12. What kind of annoying work do you most want it to take off your hands first?
   Hints:
   - repetitive
   - time-consuming
   - mechanical
   - easy to forget
   - mentally draining

13. What jargon, abbreviations, or special terms in your work must it understand?
   Hints:
   - project names
   - role names
   - team shorthand
   - industry terms

14. When it receives a task, what is the first thing it must not forget to do?
   Hints:
   - confirm the goal
   - read the material
   - list the key points
   - check whether this was done before

15. During the middle of the work, what is the most important step it must get right?
   Hints:
   - do not miss the key point
   - do not confuse the audience
   - do not invent facts
   - sort priority first

16. Before giving work back to you, what must it always check?
   Hints:
   - typos
   - logic
   - tone
   - format
   - missing items

17. When it hands something back to you, what fixed parts must the result include?
   Hints:
   - conclusion
   - summary
   - table
   - next steps
   - open questions

18. If your material is incomplete, should it ask for more first, or try making a draft anyway?
   Hints:
   - ask first
   - draft first
   - simple things draft first, key missing data must be asked for

19. If it realizes halfway through that the direction may be wrong, should it keep going or stop and check with you?
   Hints:
   - stop as soon as unsure
   - finish the rough version first
   - small mistakes it can fix itself, big direction changes must be confirmed

20. What kinds of things may it do directly, and what kinds must it ask you about first?
   Hints:
   - organizing can be direct
   - external sending must ask first
   - pricing and commitments must ask first
   - critical file changes must ask first

## Stage 3: Maintain the Brain

Use these when shaping retrieval, memory, recovery, and long-term maintenance behavior.

21. When it is unsure and needs information, should it search first or ask you first?
   Hints:
   - search first
   - ask first
   - simple things search first, complex things ask first

22. If it searches online, what sources should it trust first?
   Hints:
   - official websites
   - major media
   - industry articles
   - community experience
   - your preferred sites

23. Are there any kinds of sites or content you would rather it never cite or rely on?
   Hints:
   - content farms
   - clickbait
   - low-quality aggregators
   - communities you do not trust

24. Should it quietly remember useful information by default, or tell you before saving it?
   Hints:
   - remember by default
   - tell me first
   - only important things should be saved

25. Before answering professional questions, should it look at previous remembered context first or just answer from the current conversation?
   Hints:
   - check memory first
   - answer directly
   - important topics check memory first, small things answer directly

26. When it says "I remembered that" or "I will do it like this next time", how should it prove that it really saved it?
   Hints:
   - tell me where it wrote it
   - show a short saved summary
   - never claim memory unless it actually wrote it

27. After a gateway restart, if it sends a recovery notice, what style should that notice be?
   Hints:
   - one short sentence
   - include a short recovered summary
   - do not notify me unless something is wrong

28. After you start a new conversation with `/new`, how should restored short-term context feel?
   Hints:
   - restore silently
   - say one short line that context was restored
   - show me the recovered key points first

29. If it sees that you are teaching the same thing repeatedly, or it keeps making the same mistake, what should it do?
   Hints:
   - suggest turning it into a default rule
   - save it silently as a candidate rule first
   - wait until it happens several times before raising it

30. What counts as "too proactive" or "too annoying" for you?
   Hints:
   - interrupting too often
   - reminding me about small things
   - reporting every tiny memory write
   - upgrading rules before I confirm

## Usage Guidance

### Default mode

Use Stage 1 first.
Then move into Stage 2 inside real work.
Use Stage 3 for maintenance preferences and long-lived behavior.

### Full questionnaire mode

Only send the full 30-question set when the user explicitly asks for:

- the complete questionnaire
- a one-shot rebuild form
- a worksheet for themselves or clients
